Privacy-First Architektur: Die UI zeigt standardmäßig nur aggregierte Daten (Kategorien, Summen, Trends). Einzeltransaktionen, Kontobewegungen und Details sind nur für HA-Admins sichtbar. Normale User sehen ausschließlich vorgefertigte Zusammenfassungen.

Entity-Mapping: Jedes Bankkonto = 1 Sensor (mit Bank-Logo via GoCardless). Gesamtsaldo optional (Default off). Budget-Limits als Number-Entities, Split-Modell als Select-Entity — alles Dashboard-steuerbar. Transaktionen werden als Sensor-Attribute gecached (nicht als eigene Entitäten).

Monatszyklus-Logik: Recurring-Transaktionen (Miete, Kredite, Abos) werden ihrem "logischen Monat" zugeordnet, auch wenn das Buchungsdatum durch Banktage abweicht. Das verhindert Doppel-Buchungen in einem Monat und Null-Buchungen im nächsten.

Bonus-Erkennung: Einnahmen ≥15% über dem 3-Monats-Durchschnitt lösen eine HA-Notification aus. Der User bestätigt ob es ein Bonus ist. Bestätigt: Überschuss geht ins Spielgeld, nicht in die monatliche Balance-Berechnung. Schwelle konfigurierbar in den Settings.
